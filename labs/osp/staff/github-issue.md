# Github Issue Template

Title: Improving the ease of contribution for beginners

Dear maintainers,

We're reaching out on behalf of an ongoing partnership between California State University Monterey Bay's [Computing Talent Initiative](https://computingtalentinitiative.org/) and [CodeDay](https://www.codeday.org/), a nonprofit organization. We're working on a program for college students who are new to contributing to open-source software, and we'd love to invite them to help closing some tickets in your project! 

One of the goals of our program is to get more students to become comfortable with contributing to larger open source projects. We talked with the maintainers of a few projects, and they suggested the following general improvements to help making a repository more contributor-friendly:

- [ ]  Documentation for setting up a local developer environment
- [ ]  Having an [Architecture.md file](https://matklad.github.io/2021/02/06/ARCHITECTURE.md.html) that helps with understanding the codebase
- [ ]  “good first issue” labels in Issues
- [ ]  CI set up with linting and tests, that is also reproducible on local

If possible, we'd also love to invite you to record a short walkthrough (or host a synchronous webcast) to help students establish a mental modal of your codebase. If you're interested in hosting something like this, please feel free to get in touch with me at mingjie@codeday.org!
